# FileRoll

English · [中文](docs/README.zh.md) · [日本語](docs/README.ja.md) · [Español](docs/README.es.md)

FileRoll is a personal cloud storage application with WebDAV support, offering file management, version control, sharing, multi-user management, and more. Built with PHP 8+ and SQLite/MySQL, it can be accessed via a browser or any WebDAV client.

## Features

- **File & Folder Management**: drag-and-drop upload, move, copy, rename, trash, starred items
- **Version Control**: save and one-click restore file history versions
- **Sharing**: generate public / password-protected / time-limited share links
- **Multi-user & Permissions**: administrators can manage users, quotas, and roles
- **WebDAV Support**: compatible with Windows Explorer, macOS Finder, Cyberduck, RaiDrive, and other clients
- **Security Hardening**: bcrypt password hashing, CSRF protection, path traversal protection, XSS filtering, rate limiting
- **Internationalization**: supports 8 UI languages
- **CLI Management**: built-in commands for migrations, user creation, password reset, cleanup tasks, etc.

## Tech Stack

- PHP 8.0+
- SQLite / MySQL / MariaDB
- Composer dependency management
- PSR-4 autoloading, MVC architecture
- nginx / Apache deployment

## Quick Start

### Docker Deploy (Recommended)

The fastest way to get started — no need to install PHP, Nginx, or any dependencies:

```bash
docker run -d --name fileroll \
  -p 80:80 \
  -v fileroll_storage:/var/www/fileroll/storage \
  -v fileroll_config:/var/www/fileroll/config \
  -e APP_URL=https://yourdomain.com \
  --restart unless-stopped \
  ghcr.io/laingyulee/fileroll:latest
```

See [DOCKER.md](docs/DOCKER.md) for Docker Compose, MySQL, reverse proxy, and more.

### Manual Deploy

#### Requirements

| Item | Requirement |
|---|---|
| PHP | >= 8.0 |
| Extensions | PDO, pdo_sqlite/pdo_mysql, json, mbstring, session, ctype, filter, fileinfo, gd |
| Web Server | nginx or Apache (mod_rewrite) |
| Database | SQLite (default) or MySQL 5.7+ / MariaDB 10.3+ |

#### One-Minute Deploy

```bash
git clone https://github.com/laingyulee/fileroll.git fileroll
cd fileroll
composer install --no-dev
mkdir -p storage/content storage/temp storage/trash
chmod -R 775 storage/ config/
```

Then configure your web server to point to `public/` (recommended) or the project root (LNMP approach), and visit the domain to enter the installation wizard.

> **Detailed deployment instructions** (including complete nginx/Apache configuration, LNMP one-click package, permissions, FAQ) can be found in [DEPLOYMENT.md](DEPLOYMENT.md). For Docker deployment, see [DOCKER.md](docs/DOCKER.md).

### Installation Wizard

Visit `https://yourdomain.com/` in your browser and follow the guided steps:

1. Environment check
2. Database configuration (SQLite or MySQL)
3. Create administrator account
4. Complete installation

After installation, it is recommended to delete the `install/` directory:

```bash
rm -rf install/
```

## WebDAV Usage

FileRoll provides a standard WebDAV endpoint:

```
https://yourdomain.com/dav
```

After logging in with Basic Auth, you can manage cloud files just like a local disk. Supports large file chunked uploads and partial synchronization protocols compatible with Nextcloud/ownCloud clients.

## Security Notes

- Passwords are stored using bcrypt (cost=12)
- All form submissions validate CSRF tokens
- Files are stored by content hash to avoid duplication and support version rollback
- Uploaded file names and paths are strictly sanitized to prevent path traversal
- WebDAV uploads can only be performed by the authenticated user themselves; cross-user access is prohibited
- WebDAV HTML browser plugin and debug logging are disabled by default in production
- Detailed security fix records can be found in commit history and the server configuration chapter of [DEPLOYMENT.md](DEPLOYMENT.md)

## Configuration

All configurations are centralized in `config/config.php`, generated by the installation wizard. Key items:

```php
'app' => [
    'url' => 'https://fileroll.yourdomain.com',
    'debug' => false,          // Must be disabled in production
],
'session' => [
    'cookie_params' => [
        'secure' => true,      // Enable in HTTPS environment
        'httponly' => true,
        'samesite' => 'Lax',
    ],
],
```

For more configuration options, refer to `config/config.sample.php`.
